## Goal

This guide explains how to document a basic IT support issue before attempting a fix. The purpose is to make troubleshooting easier, cleaner, and more professional.

## Steps

1. Write down the device type, OS, and when the issue started.
2. Take a screenshot or photo of the error message.
3. Record what the user was doing before the issue happened.
4. Check simple items first, including power, Wi-Fi, updates, and account login status.
5. Save the notes so the issue can be tracked if it happens again.

## Audio-Visual Use

The written steps explain the process. The screenshot or photo gives visual proof of the problem. Captions explain what the viewer should notice in the image. This makes the guide useful for clients, students, and anyone who needs a clear support process.
