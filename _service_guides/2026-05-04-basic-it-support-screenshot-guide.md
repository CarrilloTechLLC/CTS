### Goal

This guide explains how to document a basic IT support issue before trying to fix it. The goal is to make the support process cleaner, easier to follow, and more professional. Good documentation helps track what happened, what was checked, and what steps were already taken.

### Steps

1. Write down the device type, OS, and when the issue started.
2. Take a screenshot or photo of the error message.
3. Record what the user was doing before the issue happened.
4. Check simple items first, including power, Wi-Fi, updates, and account login status.
5. Document each troubleshooting step and the result.
6. Save the notes so the issue can be tracked if it happens again.

### Why This Matters

Basic IT support is not only about fixing the problem. It is also about documenting the issue in a way that makes sense later. When the problem is written down clearly, another technician can understand what happened without starting over. It also helps clients or users see that the issue was handled in an organized way.

### Result

By following this process, the support issue becomes easier to track, easier to explain, and easier to fix. The screenshot supports the written guide by showing a clear example of what a troubleshooting process can look like on a computer screen.
